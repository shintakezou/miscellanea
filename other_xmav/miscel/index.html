asciilist.sh

This is a Bash script. It happens (it happened to me) you need to write specific ASCII bytes to a file or to a pipe. This script does it.

asciilist.sh 0x63 0x69 0x71 0x6f 0x0a

For example writes to output "ciao" (followed by a LineFeed). Decimal allowed too.

dothumb.pl

A Perl wrapper to imlib's create_scaled_image. You give a file as first argument (the file must be an image!) and a height in pixel. You obtain a scaled image file with a _th (thumb) added.

dothumb.pl navastro.png 256

This scales the navastro.png image file to a height of 256 pixels. Beware of the fact the imlib recognises filetype by extension.

mgs2

This is a rather complex Bash script. What is its purpose? Why does it exist? I can't remember if it was on Phreak or BFI or something like that, however there they used to write articles with embedded codes (e.g. in C), and provided a program that get those codes...

My mgs2 is a Bash script that does this: parse file, extracting regions of it according to preset identifiers. I can't remember details because 1) I wrote it two years ago and 2) I've never used it! (But it seems I tested it and it worked...)

It uses all sort of (GNU) tools: sed, egrep, awk, and of course common *n*x tools like mkdir, mv, cat and so on... All the shell stuffs... It does more: when getting C sources, it can append them for compiling! (gcc is used); then, each embedded file has its own path and this is created and so on...

A little bit confused? Yes I am too. Why doesn't it use mkdir -p? Or why...? Well, then specific command per file can be executed, or the compilation of C sources can be avoided...

mgs2 FILE

You maybe need an example to test on it and try to understand... So I give you an example here. First create a dir where you want your file, since current dir is used and you likely don't want mgs messes it all up (?). Do something like (from your preferred shell)

$ mkdir testdir && cd testdir
$ path/to/mgs2 path/to/example.txt

Where $ is the prompt so you know when you have to hit return. If everything's fine, you will find 5 files: helloworld.c and its executable (if compiling was successful), estratto.txt, code.c and lesson.txt.

nocr.pl

It strips CR (Carriage Return) from a file/stream (it uses piping so you can use it as a filter).

nocr.pl <file.txt >filenocr.txt

MS DOS and similar systems use the sequence of byte CR LF as newline (in decimal the bytes' values are 13 10). Stripping the CR gives the *n*x/Amiga convention for newline (just a single LF). If you are dealing with Mac files (before of Mac OS X?) the newline is the CR alone, so stripping it means creating a single long line.
